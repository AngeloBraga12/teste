Fundamentos Cartilha Principios Noções de Segurança Vulnerabilidades

Fundamentos de técnicas de ataque e identificação de vulnerabilidades

Neste site, abordaremos os fundamentos das técnicas de ataque e identificação de vulnerabilidades em sistemas de informação.

Entenderemos os conceitos básicos de segurança da informação, as diferentes técnicas de ataque usadas por invasores e as metodologias para identificar vulnerabilidades em sistemas.

Também discutiremos as medidas de segurança que podem ser implementadas para proteger sistemas de informação contra ataques.



Footpoint

Footpoint: O que é?

Footpoint é um termo usado em geometria para se referir ao ponto em que uma linha ou curva intercepta uma superfície.

No contexto da segurança da informação, o footpoint é o ponto em que um atacante consegue estabelecer uma conexão com um sistema alvo.

O footpoint pode ser usado pelos atacantes para lançar uma variedade de ataques, como ataques de negação de serviço, ataques de engenharia social e ataques de injeção de SQL.

Descoberta de Informações: Varredura/Análise

A varredura e análise de informações é uma etapa importante do processo de descoberta de informações.

Nesta etapa, os testadores de penetração usam uma variedade de ferramentas e técnicas para coletar informações sobre o sistema alvo.

Varredura de rede:

A varredura de rede é uma técnica usada para identificar hosts ativos em uma rede.

Os testadores de penetração usam ferramentas de varredura de rede para identificar hosts que estão vulneráveis a ataques.

Análise de portas:

A análise de portas é uma técnica usada para identificar os serviços que estão sendo executados em um host.

Os testadores de penetração usam ferramentas de análise de portas para identificar portas que estão abertas e vulneráveis a ataques.

Coleta de informações:

A coleta de informações é uma técnica usada para coletar informações sobre o sistema alvo, como o sistema operacional, as versões de software e as configurações de segurança.

Os testadores de penetração usam uma variedade de ferramentas e técnicas para coletar informações sobre o sistema alvo.



Enumeração

Enumeração: O que é?

A fase de enumeração (enumeration), sendo a terceira etapa do processo de teste de intrusão, envolve estabelecer uma conexão ativa com o sistema alvo para realizar consultas direcionadas. Estas consultas visam coletar informações detalhadas sobre a infraestrutura de rede e os sistemas em questão.

Testes de Penetração e Testes de Vulnerabilidades

Testes de penetração e testes de vulnerabilidades são dois tipos de testes de segurança que podem ser usados para identificar e corrigir vulnerabilidades em sistemas de informação.

Testes de Penetração

Os testes de penetração são um tipo de teste de segurança que simula um ataque real a um sistema de informação. Os testadores de penetração usam uma variedade de técnicas para tentar obter acesso não autorizado a um sistema, incluindo:

  • Exploração de vulnerabilidades de software.
  • Ataques de phishing.
  • Ataques de engenharia social.

Os resultados dos testes de penetração podem ser usados para identificar e corrigir vulnerabilidades em sistemas de informação.

Testes de Vulnerabilidades

Os testes de vulnerabilidades são um tipo de teste de segurança que identifica vulnerabilidades em sistemas de informação. Os testadores de vulnerabilidades usam uma variedade de ferramentas e técnicas para identificar vulnerabilidades, incluindo:

  • Scanners de vulnerabilidades.
  • Analisadores de código fonte.
  • Testes manuais.

Os resultados dos testes de vulnerabilidades podem ser usados para identificar e corrigir vulnerabilidades em sistemas de informação.



Engenharia Social

Engenharia Social: O que é?

Engenharia social é um tipo de ataque cibernético em que os atacantes usam técnicas de manipulação psicológica para enganar as vítimas a revelar informações confidenciais ou realizar ações que podem comprometer a segurança de seus sistemas.

Os atacantes podem usar uma variedade de técnicas de engenharia social, como phishing, vishing e smishing, para enganar as vítimas.

O objetivo dos atacantes é obter acesso a informações confidenciais, como senhas, números de cartão de crédito ou informações pessoais.

Os atacantes também podem usar engenharia social para enganar as vítimas a instalar malware em seus computadores ou a realizar outras ações que podem comprometer a segurança de seus sistemas.

Exemplos de engenharia social:

  • Um atacante pode enviar um e-mail de phishing para uma vítima, fingindo ser um banco ou outra organização legítima. O e-mail pode conter um link para um site falso que se parece com o site legítimo. Se a vítima clicar no link e inserir suas credenciais de login, o atacante poderá obter acesso à sua conta bancária.
  • Um atacante pode ligar para uma vítima, fingindo ser um funcionário do suporte técnico. O atacante pode pedir à vítima para instalar um software malicioso em seu computador ou para fornecer informações confidenciais.
  • Um atacante pode enviar uma mensagem de texto para uma vítima, fingindo ser uma empresa de entrega. A mensagem pode conter um link para um site falso que se parece com o site legítimo. Se a vítima clicar no link e inserir suas informações pessoais, o atacante poderá obter acesso a essas informações.

Como se proteger:

  • Não clique em links em e-mails ou mensagens de texto de remetentes desconhecidos.
  • Não forneça informações confidenciais por telefone ou e-mail.
  • Use um software antivírus e um firewall.
  • Atualize seu software regularmente.
  • Seja cauteloso ao compartilhar informações pessoais online.


Negação de Serviço

DoS & DDoS attacks

Ataques DoS (Distributed Denial of Service) são um tipo de ataque cibernético em que os atacantes inundam um sistema alvo com tanto tráfego que ele fica indisponível para usuários legítimos. Um ataque de negação de serviço (DoS) inunda um servidor com tráfego, tornando um site ou recurso indisponível. Um ataque de negação de serviço distribuído (DDoS) é um ataque de DoS que usa vários computadores ou máquinas para inundar um recurso alvo.

Como funciona:

Os atacantes usam uma rede de computadores infectados (chamada de botnet) para enviar um grande volume de tráfego para o sistema alvo.

Esse tráfego pode ser composto por solicitações HTTP, pacotes TCP ou outros tipos de tráfego.

Quando o sistema alvo recebe uma grande quantidade de tráfego, ele fica sobrecarregado e não consegue atender às solicitações legítimas.

Objetivo dos atacantes:

  • Tornar um site ou serviço indisponível para usuários legítimos.
  • Extorquir dinheiro das vítimas, ameaçando realizar ataques DDoS se elas não pagarem.
  • Usar ataques DDoS como forma de protesto ou ativismo.

Exemplos de ataques DDoS:

  • Em 2016, o site da Dyn, uma empresa de serviços de DNS, foi derrubado por um ataque DDoS que durou várias horas.
  • Em 2017, o site do governo brasileiro foi derrubado por um ataque DDoS que durou vários dias.
  • Em 2018, o site do Facebook foi derrubado por um ataque DDoS que durou várias horas.

Como se proteger:

  • Use um serviço de proteção contra DDoS.
  • Configure seu firewall para bloquear tráfego suspeito.
  • Limite o número de conexões que podem ser feitas ao seu sistema.
  • Monitore seu sistema para detectar e mitigar ataques DDoS.

SQL injection

SQL injection é um tipo de ataque cibernético em que os atacantes injetam código SQL malicioso em um aplicativo da Web. Esse código pode ser usado para roubar dados, modificar dados ou até mesmo obter acesso ao sistema operacional subjacente.

Como funciona:

Os atacantes usam vulnerabilidades em aplicativos da Web para injetar código SQL malicioso. Esse código pode ser inserido em formulários da Web, campos de pesquisa ou outros pontos de entrada de dados.

Quando o aplicativo da Web executa o código SQL malicioso, ele pode acessar dados confidenciais, modificar dados ou até mesmo obter acesso ao sistema operacional subjacente.

Objetivo dos atacantes:

  • Roubar dados confidenciais, como senhas, números de cartão de crédito ou informações pessoais.
  • Modificar dados, como alterar saldos de contas bancárias ou excluir registros de clientes.
  • Obter acesso ao sistema operacional subjacente para instalar malware ou realizar outras atividades maliciosas.

Exemplos de SQL injection:

  • Em 2013, o site da Sony Pictures foi hackeado por um ataque de SQL injection que roubou informações pessoais de mais de 100 milhões de usuários.
  • Em 2014, o site do governo brasileiro foi hackeado por um ataque de SQL injection que roubou dados pessoais de mais de 1 milhão de usuários.
  • Em 2015, o site do Facebook foi hackeado por um ataque de SQL injection que roubou dados pessoais de mais de 10 milhões de usuários.

Como se proteger:

  • Use parâmetros preparados para todas as consultas SQL.
  • Valide a entrada do usuário para evitar que código SQL malicioso seja inserido.
  • Use um firewall de aplicativos da Web para bloquear ataques de SQL injection.
  • Mantenha seus aplicativos da Web atualizados com os patches de segurança mais recentes.

Ransomware

Ransomware é um tipo de malware que criptografa os arquivos da vítima e exige o pagamento de um resgate para descriptografá-los.

Como funciona:

Os atacantes usam uma variedade de métodos para infectar computadores com ransomware, incluindo e-mails de phishing, downloads drive-by e vulnerabilidades de software.

Uma vez que o ransomware está instalado no computador da vítima, ele criptografa os arquivos da vítima e exige o pagamento de um resgate para descriptografá-los.

Objetivo dos atacantes:

  • Extorquir dinheiro das vítimas, ameaçando excluir ou publicar seus arquivos se elas não pagarem o resgate.
  • Usar ransomware como forma de protesto ou ativismo.

Exemplos de ransomware:

  • Em 2017, o ransomware WannaCry infectou mais de 200.000 computadores em todo o mundo, exigindo um resgate de US$ 300 em bitcoins.
  • Em 2018, o ransomware NotPetya infectou mais de 10.000 computadores em todo o mundo, exigindo um resgate de US$ 10.000 em bitcoins.
  • Em 2019, o ransomware Ryuk infectou mais de 1.000 computadores em todo o mundo, exigindo um resgate de US$ 150.000 em bitcoins.

Como se proteger:

  • Faça backup de seus arquivos regularmente.
  • Use um antivírus e um firewall.
  • Atualize seu software regularmente.
  • Não abra e-mails de phishing.
  • Não baixe arquivos de fontes desconhecidas.
